# 데이터베이스 개요

## < 데이터와 정보 >

### 데이터

- Data
- 현실 세계에서 측정하고 수집한 사실이나 값
- 가공되지 않은 그대로의 객관적인 사실
- ex) 회원 가입 내역. 매장의 매출 내역, 고객이 로그인한 시간 등

### 정보

- Information
- 목적에 따라 데이터를 처리하여 의미를 부여한 것 (데이터를 가공한 것)
- 의미를 부여한 데이터로서 의사 결정 문제에 대한 해결의 수단으로 사용
- ex) 가입 고객의 성별/연령 분포, 매장별 평균 매출액, 베스트 셀러 등

### 데이터와 정보 개념 정립이 중요한 이유

#### 1 정확한 모델링을 위해
- 데이터를 정보로 바꾸는 구조를 이해해야 현실 세계를 데이터베이스로 정확히 표현 가능
	- 고객, 상품 등은 단순한 데이터이지만, 이를 연결하면 유용한 정보
	- 데이터베이스 설계의 지침
#### 2 데이터 처리의 효율성 확보
- 어떤 데이터가 가치 있는 정보가 될 수 있을 지 판단하여, 불필요한 데이터 제거 및 정확한 설계가 가능
#### 3 의사결정 지원
- 정보는 의사결정을 하기 위한 기반으로서, 정보를 생성하는 데이터를 이해해야 정확한 결과를 도출 가능
#### 4 보안과 무결성 확보
- 어떤 데이터가 중요한 정보로 쓰이는지 구분하면, 보호해야할 데이터와 그렇지 않은 데이터를 구분 가능

## < 데이터의 분류 >

### 정형 데이터(Structured Data)

- 구조가 정해진 데이터
- 표(테이블) 같은 고정된 형태로 저장
- ex) 엑셀 스프레드 시트, CSV 파일, RDBMS 등

### 비정형 데이터(Unstructured Data)

- 구조화 되어 있지 않은 데이터
- 고정된 형태로 표현 불가
- ex) 영상, 이미지, SNS 게시글, 텍스트 문서 등

### 반정형 데이터(Semi-Structured Data)

- 내부 정보를 표현하는 '메타데이터' 가 포함
- 형식이 존재하진 않지만, 특정 데이터를 설명하는 설명서나 메타 데이터를 저장
- ex) XML, JSON, 웹 로그 파일 등

## < DataBase 의 개념 >

### DataBase

- 특정 목적을 위해 관련 있는 데이터들을 구조적으로 모아 놓은 저장소


			저장 데이터 관리자 ---- 물리적 공간 관리 ---|
											  v
\[사용자]-- 질의(SQL) -->\[질의처리기] -- 질의 해석 --> \[물리 저장소]
												 ^
					트랜잭션 관리자 ---- 트랙잭션 관리 ---|

#### 질의처리기(Query Proccesor)
- 사용자의 질의(SQL)를 처리하여 DataBase 에서 원하는 정보를 추출하는 역할
- SQL(Structured Query Language)
	- 국제 표준 DataBase언어
#### 트랜잭션 관리자
- 트랜잭션의 시작, 종료, 복원 등의 작업을 제어하여 데이터 일관성을 유지하는 역할
- 트랜잭션(Transaction)
	- DataBase 의 논리적인 기본 처리 단위
#### 저장 데이터 관리자
- DataBase 의 물리적 공간에 대한 관리를 통한 데이터를 효율적 저장을 지원하는 역할


#### DataBase 의 데이터 관점 특성

- 공용 데이터(Shared Data)
	- 여러 사용자가 다른 목적으로 데이터를 공동 이용
	- 동시에 여러명이 같은 데이터에 접근이 가능해야 한다
- 통합된 데이터(integrated Data)
	- 동일한 데이터가 중복되어 있지 않음
	- 동일한 데이터가 중복 저장되면 저장소 공간 낭비와 사용 금액 측면에서 불리
- 저장된 데이터(Stored Data)
	- 저장 매체에 데이터를 저장
- 변화되는 데이터
	- 새로운 데이터 추가, 수정, 삭제에도 현재의 정확한 데이터 유지


#### DataBase 의 시스템 운영적 특성

- 실시간 접근성
	- 실시간 처리에 의한 응답이 가능해야 한다
- 계속적인 변화
	- 데이터베이스의 상태는 계속적으로 변화 한다
- 동시 공용
	- 다수의 사용자가 동시에 같은 내용의 데이터 이용 가능
- 내용에 의한 참조
	- 사용자가 요구하는 데이터 내용으로 데이터를 검색
