#### \[ 제1세대 컴퓨터 ]

- 기계식 컴퓨터의 문제점 보완하기 위해 **진공관**을 이용한 **전자식 컴퓨터**
	- 기계식의 기계 부품의 복잡한 구조
	- 잦은 고장
- **금속에 열**을 가할 경우 **전자가 튀어나오는 현상**이 이용하여 **전류가 한 쪽 방향**으로 흐르도록 만든 장치
	- **관 내부를 진공**으로 만들었기 때문에 진공관
	- **데이터 저장과 처리에 진공관**을 이용
	- 프로그램 언어로 **기계어**와 **어셈블리어**를 사용
- **ENIAC**(Electronic Numerical Integrator And Calculator)
	- 약 1만 8천 여개의 진공관
	- 약 30톤 무게
	- 기계식 장치보다 약 1천배 빠름
- **EDVAC**(Elctronic Discrete Variable Automic Computer)

#### \[ 제2세대 컴퓨터]

- 부피가 크고 수명이 짧은 진공관의 문제점을 해결하기 위해 개발
- **벨 연구소**에서 개발한 **트랜지스터** 이용
	- 진공관 크기의 1/200
	- 수명 또한 **반영구적**
- 고급 프로그래밍 언어 사용
	- **COBOL**
	- **FORTRAN**
- 기억 장치
	- **자기코어**
		- 대용량 기억 장치
	- **자기드럼**
- **시스템 프로그래밍** 개념 등장
	- **컴파일**
	- **소프트웨어 라이브러리** 등이 제공
- **IBM 7074**
- **CDC 1604**

#### \[ 제3세대 컴퓨터 ]

- 트랜지스터 대체하는 **집적회로**(**IC**: Intgraed Circuit)칩 사용
	- 2세대 컴퓨터에서 트랜지스터, 저항 등과 같은 부품이 회로 보드상에서 연결되게 만든 후 이를 다시 컴퓨터 시스템의 부품으로 장착했었음
		- 전체 제조 과정이 어렵고 비용이 많이 소모됨
	- 이를 하나의 칩으로 구성
		- 전기 회로를 구성하는 **트랜지스터**, **다이오드**, **저항** 등을 **한 개의 패키지**로 만든 
		  **전자소자시스템**
- IC 칩으로 인한 **컴퓨터의 소형화**
- 기억용량 증대
- **시분할 방식** 실현
- **경영 정보 시스템**에 대한 **체계가 확립**됨

#### \[ 제4세대 컴퓨터]

- 집적 회로가 수배 ~ 수천 개의 소자를 갖는 회로를 한 개의 기판에 넣는 
  **고밀도 집적회로**(**LSI**: Large Scale integrated Circuit)로 대치
	- 컴퓨터의 크기, 가격, 수행속도, 기억 용량 측면에서 혁신적인 발전을 하게 됨
- 기억장치
	- **반도체 기억 소자**
	- **가상 기억 장치 개념**이 도입
		- 주기억장치의 저장 능력기 향상
- **초대규모 집적회로**(**VLSI**: Very Large Scale integrated Circuit)로 개발
	- 현재의 컴퓨터에 사용
	- 5세대로도 분류
- LSI 와 VLSI 를 이용하여 컴퓨터의 주요 구성 요소를 **하나의 반도체 칩에 집적**시킨 
  **마이크로프로세서가 개발**됨
	- **개인용 컴퓨터가 개발**되어 보급
	- 고성능 개인용 컴퓨터 개발에도 영향
	- 컴퓨터 가격의 저렴화
		- 컴퓨터를 이용한 **업무의 효율성**을 위한 **자동화**가 이루어지기 시작
- 네트워크 연결을 위한 **컴퓨터 통신의 발달**
	- 혼자 사용하는 컴퓨터 시대의 끝
	- **전 세계 컴퓨터들이 네트워크를 통해 연결**되고 확장

#### \[ 차세대 컴퓨터 ]
- 1980 년대 초 일본이 제5세대 컴퓨터를 개발하겠다고 공표가 시발점
- **대규모 병렬 처리** 컴퓨터
	- 수천에서 수만 대의 처리 장치를 가진 컴퓨터일 것
	- 하드웨어의 부품 발달과 저렴한 가격으로 하나의 **컴퓨터 시스템 내의 부품이 중복**되어도 경계성을 유지 가능
	- **클러스터 컴퓨터 기술**을 활용
		- 단위 컴퓨팅 노드들을 **고속 클러스터 연결망**으로 묶어 **하나의 시스템으로 활용**할 수 있음
		- I/O 성능을 향상시키기 위한 InfiniBand 등의 기술 등으로 발전
		- **분산 처리**
- **양자 컴퓨터**
	- 양자의 얽힘이나 중첩 같은 **양자 역학적인 현상을 활용**하여 **연산을 수행**하는 컴퓨터
	- **한 개의 처리 장치**에서 **여러 계산을 동시**에 수행
		- 정보 처리량과 속도 면에서 지금까지의 컴퓨터에 비해 매우 뛰어남
	- **양자 병렬성**을 이용하면 **기존 슈퍼 컴퓨터**에서도 **풀기 어려운 문제를 풀 수 있으나**. 아직은 매우 제한적인 문제에 대해서만 가능
- **신경망 컴퓨터** 등
	- **인간의 신경망을 모방**한 구조를 가진 컴퓨터
		- 주로 AI 쪽에서 많이 사용
	- 기존 컴퓨터는 폰노이만의 내장 프로그램 구조를 따름
		- 순차적으로 프로그램을 수행하기 때문에 병렬 처리에 적합하지 않음
		- 이 문제를 해경하기 위해 신경망 컴퓨터에 대한 연구 진행

### < 컴퓨터의 기본 구조 >

컴퓨터는 크게 **하드웨어**와 **소프트웨어**로 나눔
**하드웨어**
- 컴퓨터를 구성하는 **물리적 기계장치**
- 보편적인 컴퓨터의 기본 구조는 하드웨어를 설명함
	- **CPU**(중앙처리장치)
		- **산술 논리 연산** 장치(**ALU**: Arithmetic Logic Unit)
		- **제어 장치**(Control Unit)
		- **레지스터**(Register)
	- **I/O**(입출력) 장치
		- 키보드
		- 마우스
		- 모니터
		- 스피커
	- **Main Memory**(주기억장치)
		- **휘발성** 기억장치
		- **비휘발성**인 **보조 기억장치** 연결 사용
	- **System Buses**(시스템 버스)
**소프트웨어**
- 하드웨어 위에서 **논리적**으로 구동하는 **프로그램**
- 컴퓨터 하드웨어를 제어하고 사용자를 위한 편의성을 제공하는 프로그램인 **운영체제**를 **별도로 분리**하는 **경우도 있음**
#### \[ 중앙 처리 장치 ]

+ **CPU**(Central Processing Unit)
+ 컴퓨터 시스템을 통제하고 프로그램 연산을 실행, 처리하는 가장 핵심적인 장치
+ **제어장치**
	+ Control Unit
	+ 각 요소들을 순서에 맞춰 작동 시키는 제어를 수행
	+ **CPU 내**에서 발생하는 **모든 동작**을 **제어하고 관리**
	+ **기억 장치에서 명령어를 읽고** 해석한 **결과에 따른 명령을 수행**하기 위한 **제어신호**를 
	  **만드는 기능**을 수행
+ **산술논리연산장치**
	+ **ALU**(Arithmetic Logic Unit)
	+ 실제 연산을 수행하는 장치
		+ 사칙연산을 포함한 **산술연산**
		+ 참 $\cdot$ 거짓을 판별하는 **논리연산** 수행
+ **레지스터**
	+ CPU 내부에 존재하는 기억장치
	+  데이터 저장(임시적 보관 기능)
		+ 연산 처리를 위한 데이터
		+ 데이터 주소
		+ 연산 결과 등
	+ **프로그램 계수기**(**PC**: Program Counter)
	+ **명령어 레지스터**(Instruction Register)
	+ **메모리 주소 레지스터**(Memory address Register)
	+ **메모리 버퍼 레지스터**(Memory Buffer Register) 등
+ 버스
	+ 중앙처리장치 **내부의 장치**들을 **연결**하는 통로
		+ 내부 버스라고도 불림
	+ 중앙처리 외부에도 버스 존재
		+ 외부 버스
	+ **주소버스**
	+ **제어버스**
	+ **데이터 버스**

#### \[ 기억 장치 ]

데이터를 저장하는 기억 장치는 특성과 역할에 따라 **캐시기억장치**, **주기억장치**, **보조기억장치**로 분류
입력 장치에서 입력된 데이터는 **이진수로 변환**되어 **기억장치에 저장**됨
기억장치는 **계층적으로 구조**되었으며, 이는 **CPU 와 기억 장치 사이의 속도 차이**를 **극복**하기 위한 구조
- **주기억 장치**(Main Memory)
	- 컴퓨터에서 CPU 가 처리할 데이터와 명령어들을 저장하는 기억장치
	- CPU 와 직접적으로 데이터를 주고 받을 수 있도록 시스템 버스로 연결
	- **RAM**(Random Access Momory)
		- **휘발성** 기억장치
		- 전원 공급이 지속되는 동안에는 데이터를 기억
			- 차단 된 경우 저장된 데이터 사라짐 
		- **SRAM**(Static RAM)
			- **플립플롭 기억소자**
				- 회로가 복잡
				- 비용이 높음
			- 전원이 공급되는 동안 데이터 유지
		- **DRAM**(Dynamic RAM)
			- **축전지**를 통해 데이터 기억
				- 축전기가 **전자를 누전**함으로써 기억된 **정보 소실 가능**
				- **주기적인 충전**이 필요
			- SRAM 에 비해 성능면이 떨어지지만, 적절한 속도와 저렴한 비용, 높은 용량
			  으로 인해 **주기억장치로 사용**됨
	- **ROM**(Read Only Memory)
		- **비휘발성** 기억장치
		- 전원 공급이 차단되더라도 이전의 데이터 기억
		- 단편적인 예
			- **부팅** 시 컴퓨터의 전원 입력 후 OS 를 부팅하기 위해 기억할 수 있는 용도로 
			  사용 
		- 기록한 데이터를 빠른 속도로 읽을 수 있음
			- 보통은 **기록 및 수정**은 **불가능**
		- 기술의 발달로 다시 쓰고 지울 수 있는 방식에 따라 종류가 나뉨
			- **MASK ROM**
				- **제조 과정**에서 미리 내용을 **기록**한 ROM
				- 사용자가 내용을 수정할 수 없음
			- **PROM**(Programmable ROM)
				- **사용자**가 **한 번마 기록**할 수 있는 ROM
			- **EPROM**(Erasable Programmable ROM)
				- 필요시 기억된 내용을 지우고 다른 내용을 기록할 수 있는 ROM
				- **강한 자외선**을 비추어 정보를 지울 수 있음
					- 특수 장비 필요로 인해 일반 사용자가 사용하긴 어려움
- **보조기억장치**
	- 컴퓨터 사용자가 현재 주기억장치에 저장한 내용을 전원을 끄고 난 이후에도 사용할 수 있도록 지원해주는 기억장치
	- **자기디스크**
	- **자기테이프**
	- **하드디스크**(HDD)
	- CD, 플로피 디스크, DVD
	- USB
	- 플래  시메모리 등
- **캐시기억장치**
	- 중앙처리장치와 주기억장치의 **속도 차이**를 **극복**하기 위하여 중간에 위치한 기억장치
	- 주기억 장치보다 속도가 빠름
	- **임시**로 데이터 **저장**
	- **위치**에 따라 **L1 ~ L3**캐시메모리로 **분류**
		- 레벨이 높을수록 처리 속도가 우수
- 시스템 버스
	- 중앙처리장치를 중심으로 컴퓨터 시스템의 구성 요소들이 데이터를 주고받는 통로
	- 기능별 구분
		- 데이터 버스
		- 주소 버스
		- 제어 버스
	- 위치별 구분
		- 내부 버스
		- 외부 버스
	- 버스의 용량에 따라 데이터를 한 번에 주고 받을 수 있는 양이 정해짐
		- ex) 8 bit 버스 -> 8bit/s
		- 버스 또한 컴퓨터 시스템의 속도를 결정하는 요인
- 입출력장치
	- 컴퓨터시스템 외부에서 내부로 입력 받거나, 내부에서 처리한 결과를 외부로 보내는 장치
	- 컴퓨터시스템과 사용자의 상호작용을 위해서 반드시 필요
	- 입력장치
		- 키보드
		- 마우스
		- 음성인식
		- 터치 패드 등 
	- 출력장치
		- 모니터
		- 프린터
		- 이어폰 등

## 이진 호환성

### < 이진 호환성 >

#### \[ 이진 호환성이란 ]

- **Binary Compatibility**
	- Binary -> **컴파일된 실행 코드**
	- 사람이 읽을 수 있는 **고급 프로그래밍 언어**로 작성된 **코드**를 **컴파일러**를 통해 
	  소스코드를 실행하여 **기계**가 **읽을 수 있는 실행코드로 생성**
	- Binay 파일의 호환성
- 이진 호환성이란 동일한 실행 코드, 즉 **일반적으로 범용 컴퓨터 기계어 코드**를 
  **서로 다른 컴퓨터 시스템**이 **실행할 수 있다**는 **속성**
- 실행 코드는 **특정 컴퓨터 시스템**의 환경에 맞게 **컴파일**됨
	- Windows 컴파일된 파일은 Linux 에서 실행되지 않음
- 하나의 컴퓨팅 환경에서 컴파일된 실행 코드가 **다른 환경**에서도 **완벽하게 실행**되는 경우 
  **두 컴퓨팅 환경**은 **이진호환**이 가능하다
	- 두 번째 환경에서 소스코드를 **재컴파일할 필요가 없음**
- 이진호환성의 중요성
	- 소스코드 재컴파일의 문제점 
		- 소스코드가 실행되는 환경이 너무 다양하다는 것
		- 재컴파일이 오래 걸리는 복잡한 애플리케이션도 존재
		- 재컴파일시 빈번한 문제 발생
		- 재컴파일시 들어가는 시간과 노력 축소 가능
	- 컴퓨팅 환경의 변화에 따라 소스코드를 **재컴파일할 필요성이 사라진다는 것**

## 소프트웨어와 하드웨어의 상호작용

### < 소프트웨어와 하드웨어 >

컴퓨터 시스템은 크게 소프트웨어와 하드웨어로 구분
하드웨어 없이 소프트웨어는 실행할 수 없고, 소프트웨어 없이 하드웨어를 작동할 수 없다
서로 불가분의 관계
#### \[ 소프트웨어 ]

- 컴퓨터 시스템을 효율적으로 운영하기 위해 개발된 모든 프로그램
#### \[ 하드웨어 ]

- 컴퓨터 시스템을 구성하는 물리적 장치들

#### \[ 소프트웨어와 하드웨어 ]

- **소프트웨어**는 하드웨어 위에서 동작하므로 **하드웨어의 물리적 한계**에 영향 받음
- 주기억장치 용량이 64KB 일 경우 주기억장치에 한 번에 적재할 수 있는 용량은 64KB 가 됨
	- 소프트웨어가 한 번에 적재할 수 있는 데이터의 용량은 저장공간인 64KB 의 
	  **물리적 한계를 넘을 수 없음**
	- '**가상기억장치**' 개념을 통해 물리적 주기억장치의 용량이 64임에도 불구하고 
	  128KB 인 것처럼 동작이 가능
		- **보조기억장치의 일부**를 소프트웨어를 통해 **주기억장치로 인식**하게 만드는 것
- 하드웨어의 성능 발전시 그에 따라 소프트웨어의 수행 능력이나 데이터의 처리량 증가
- **시스템 소프트웨어**의 **개발**로 **하드웨어에서도 처리 속도 향상**시킬 수 있음

## 시스템 소프트웨어
### < 시스템 소프트웨어 >

소프트웨어는 크게 **응용 프로그램**과 **시스템 프로그램**으로 나눌 수 있다. 
#### \[ 응용 프로그램(응용 소프트웨어) ]

- **사용자의 문제를 해결**해주는 프로그램
- 사용자가 직접 만든 프로그램도 포함

#### \[ 시스템 프로그램(시스템 소프트웨어) ]

- **응용 프로그램을 실행하기 위한 플랫폼을 제공**하고, **하드웨어를 제어**하기 위한 
  모든 소프트웨어
- 하드디스크의 데이터를 메모리로 전송, 모니터의 출력을 하는 등의 작업을 수행
- 시스템 소프트웨어의 **기능에 따른 분류**
	- ##### 제어 프로그램
		- 기능 면에서 **컴퓨터시스템의 모든 동작 및 상태**를 **관리**하고 **감독**하는 프로그램
		- **감시 프로그램**
		- **작업 제어 프로그램**
			- 작업 이행 및 다른 작업으로의 이행을 자동으로 수행
			-  한 프로세스에서 다른 프로세스 전환 시 필요할 수 있는 명령어 위치, 
			  필요한 데이터, 연산 등을 자동으로 넘겨주는 등의 역할
		- **데이터 관리 프로그램**
			- 보조기억장치에 있는 데이터를 주기억 장치에 옮기기 등
	- ##### 처리 프로그램
		- **제어 프로그램의 지시**를 받아 **사용자가 요구한 문제**를 **해결**하기 위한 프로그램
		- **언어 번역기**
			- 소스(원시) 파일을 목적 프로그램이나 실행 프로그램으로 변환 등 
		- **서비스 프로그램**
			- 시스템 소프트웨어 중에서 **시스템 소프트웨어 처리를 위해서 자주 사용**하는
			  프로그램들
			- 연결 편집기
			- 정렬 프로그램 등
		- **문제 프로그램**
			- 문제 해결을 위해서 사용자가 작성한 프로그램 등 
	- ![[Pasted image 20250829102410.png]]
	
- 시스템 소프트웨어 **용도에 따른 분류**
	- ##### 운영체제
		- 사용자가 컴퓨터를 원활히 사용할 수 있는 환경
		- **하드웨어를 관리**하고 **응용 소프트웨어를 실행**하기 **위한 하드웨어 추상화 플랫폼**,
		  **공통 시스템 서비스 등**을 제공
		- **Windows, UNIX/Linux, MacOS**
	- ##### 언어번역 프로그램
		- 프로그래머가 작성한 프로그램을 실행할 수 있도록, 컴퓨터가 처리할 수 있는 형태의 언어인 **기계어로 번역하는 프로그램들**의 총칭
		- **어셈블러, 컴파일러, 인터프리터**
	- ##### 시스템 유틸리티
		- 디스크 관리, 파일 압축 등과 같이** 컴퓨터시스템을 효율적으로 사용**하기 위해 제공되는 프로그램
		- **정렬, 디스크관리, 파일압축**

### < 어셈블러 >

#### \[ 어셈블러 ]

- 어셈블리어로 작성된 프로그램을 입력 받아 실행에 적합한 형태의 목적 프로그램으로 변환하는 언어 번역 프로그램
- 어셈블리어로 작성된 원시 프로그램은 어셈블러를 통해 기계어 형태인 목적 프로그램으로 변화하는 어셈블 과정을 거쳐야 함
	- 어셈블 과정은 보통 2단계
		- 1단계 : 심볼 테이블을 생성
			- 어셈블러 테이블에 기록
			- 프로그램 첫 부분의 메모리 주소를 기준으로 각 줄의 메모리 주소를 알아낼 수 있음
			- LABEL 에 대응하는 메모리 주소를 알아낸 뒤 2 단계로 진행
		- 2단계 : ISA 명령어로 변환
			- 앞서 기록한 심볼 테이블의 정보를 바탕으로 어셈블리 코드를 
			  0과 1로 이루어진 ISA 명령어로 변환
- 어셈블러 종류
	- 단일 패스 어셈블러
		- 원시 프로그램의 하나의 명령문을 읽는 즉시 기계어로 번역하여 목적 프로그램으로 변환
	- 이중 패스 어셈블러
		- 2단계로 나누어서 수행하는 어셈블러(위 설명 참고)
	- 크로스 어셈블러
		- 현재 사용하는 컴퓨터와는 다른 명령 형태로 동작하는 컴퓨터에서 사용할 프로그램으로 변환할 때 사용하는 프로그램
- 어셈블러는 어셈블 하는 과정에서 다양한 정보를 활용
	- 이러한 정보들을 담는 공간을 '테이블'이라고 한다
	- 기계 명령어 테이블(MOT: Machine Operating Table)
		- 어셈블리어 실행 명령에 대응하는 기계어에 대한 정보를 가진 테이블
		- 어셈블리어와 기계어를 1대1 매칭시킨 테이블
	- 의사 명령어 테이블(POT: Pseudo Operating Table)
		- 의사명령과 그 명령을 처리하는 실행 루틴의 주소를 가지고 있는 테이블
	- 심볼 테이블(ST: Symbol Table)
		- 원시 프로그램의 LABEL 부분에 있는 기호들을 저장하는 테이블
		- 1 단계의 경우 LABEL 과 주소 매칭
		- 2 단계의 경우 LABEL 과 데이터 값이나 연산 명령어 매칭 
	- 리터럴 테이블(LT: 리터럴 테이블)
		- 원시 프로그램의 Operand 부분에 있는 Literal 들을 차례로 작성, 저장하는 테이블

### < 컴파일러 >

#### \[ 컴파일러 ]

- **프로그래밍 언어로 작성된 프로그램**을 컴퓨터에서 실행하기 위해 일정한 규칙에 따라 **기계어의 형태로 번역을 수행**하는 시스템 소프트웨어
- 프로그래밍 언어는 사람이 이해하는 언어이기 때문에 컴퓨터의 해석 위해 기계어 변환 작업이 반드시 필요
- 컴파일러는 어휘분석, 구문 및 의미 분석, 코드 생성 단계를 거친다
	- ##### 어휘 분석
		- 소스코드를 읽어 프로그램 문장을 구성하고 있는 최소 단위의 어휘들을 떼어내어 
		  올바른지 분석하는 단계
			- **스캐닝**(Scanning) 이라고 부르기도 함
		- 검출된 어휘들을 참조하여 Token 을 생성함
			- Token 은 **토큰 이름**, **속성 값 구조**를 가지고 있음
	- ##### 구문 분석
		- 문장 구조가 올바른지 분석하는 단계
		- 어휘 분석 단계에서 만들어진 Token 을 이용하여 소스코드의 문법 구조를 서술하는 구문 트리(Syntax Tree) 생성
	- ##### 의미 분석
		- 구문 분석의 결과에 따른 의미 및 기능을 분석하고 기능이 올바르게 
		  수행될 수 있도록 환경을 조성하는 단계
		- 구문 분석 단계에서 만들어진 구문 트리와 심볼 테이블 내 정보를 이용하여 
		  소스코드가 의미적으로 부합하는지 검사
	- ##### 중간 코드 생성 단계
		- 구문 분석 단계에서 만들어진 결과를 이용하여 코드를 생성
	- ##### 코드 최적화 단계
		- 코드를 좀 더 효율적으로 만들어 실행 시 언어의 성능이나 자원 소모를 
		  결정 짓는 중요한 단계
	- ##### 코드 생성 단계
		- 이전 단계를 통해 분석된 소스코드를 기계에 맞는 어셈블리어나 기계어로 변환

#### \[ 링커 ]

- **연결 편집기**라고도 불림
- 언어번역 프로그램인 생성한 **목적 프로그램**들, 라이브러리, 또 **다른 실행 프로그램 등**을 
  **연결**하여 **실행 프로그램을 만드는 소프트웨어**
- ##### 링킹
	- 링커가 **여러 프로그램을 연결하는 과정**
- 소프트웨어 개발에서 **독립적인 컴파일을 가능하게** 하는 중요한 역할을 담당
- 큰 규모의 응용 프로그램을 하나의 원시 프로그램이 아닌 **여러 개의 원시 프로그램으로 모듈화**하여 별도로 수정
- **하나의 모듈을 수정**하는 경우 **해당 모듈만 다시 컴파일**하여 링킹을 수행하는 방식으로 
  변경 작업을 적용할 수 있다
- ##### 정적 링킹
	- 실행 프로그램을 만들 때 프로그램에서 사용하는 모든 라이브러리 모듈을 복사하여 연결하는 방식
	- 정적 링킹으로 만들어진 프로그램은 라이브러리 등이 중복되기 때문에 크기가 큼
		- 두 프로그램이 동일한 라이브러리 필요시 각각 해당 라이브러리를 복사해서 소유함
		- 비효율적인 메모리를 사용
	- 동적 링킹에 비해 프로그램 수행 속도가 빠름
	- 모든 프로그램이 하나의 실행 프로그램으로 만들어지기 때문에 프로그램의 불일치에 대한 우려가 없음
- ##### 동적 링킹
	- 실행 프로그램을 만들 때 프로그램에서 사용하는 모든 라이브러리를 모듈에 복사하지 않고 해당 모듈의 주소만을 가지고 연결하는 방식
	- 주소만 가지고 있어 효율적인 메모리 사용
		- 한 번 적재된 라이브러리는 메모리에 그대로 있기 때문에 다른 프로그램이 다시 
		  사용할 수 있음
	- 메모리 주소를 찾아가는 방식으로 오버 헤드가 존재하기 때문에 정적 링킹에 비해 느림
	- 프로그램 불일치가 발생할 수도 있음

#### \[ 로더 ]

- **컴퓨터 내부로 정보를 들여**오거나 **로드 모듈**을 **하드디스크 등의 보조기억장치에서 주기억장치로 적재**하는 소프트웨어
- 기본적으로 할당, 연결, 재배치, 적재의 기능을 수행 하지만 각 기능을 
  언어번역 프로그램 혹은 링커 등 **시스템 소프트웨어가 수행할 수도 있음**  
	- ##### 할당(Allocation)
		- 프로그램을 실행하기 위해 **주기억장치 내 공간을 확보**
	- ##### 연결(Linking)
		- 부프로그램 호출 시 **부프로그램이 할당된 기억 장소**의 **시작 주소**를 **호출한 부분에 등록**하여 **연결**
	- ##### 재배치(Relocation)
		- **보조기억장치에 저장된 프로그램**이 사용하는 각 **주소**들을 **할당된 실제 주소**로
		  **배치**시키는 기능
	- ##### 적재(Loading)
		- **실행 프로그램**을 할당된 **주기억장치의 공간에 실제로 옮김**
