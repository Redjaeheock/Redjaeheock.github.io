## 데이터베이스 개요
### < 데이터베이스의 개념 >

4차 산업 이래 방대한 정보 발생
정보들 속에서 원하는 정보를 찾는데 불편함 가.
지금의 사회는 정보를 효율적으로 검색하고 분석하여 새로운 가치를 창출하는 것이 더 
중요시되고 있음
조직에서 중요할 결정을 내리기 위해선 자료를 수집하고, 처리, 분석해서 올바른 정보를 만들어야 한다.
컴퓨터를 사용하여 데이터를 수집하고 분석하는 데 정보 처리 기술이 활용됨
( 우리나라에선 현재 마이데이터 법을 통해서 데이터 3법이 통과 되면서 우리 데이터들이 활용되고 있으나, 개인 정보에 대해 민감한 부분으로 얽혀 있음)

#### \[ 자료와 정보 ]

- **자료**(Data)
	- 현실 세계에서 **관찰이나 측정**을 통해 **수집**한 **단순한 사실**이나 **값**
	- **가공되지 않은 상태**
	- 숫자로 표현되는 **수치 데이터**와 문자로 표현되는 **string**, **text**, **image**, **graphic** 등을 포함
- **정보**(information)
	- **의사 결정에 도움**을 줄 수 있는 유용한 형태
	- **자료를 가공**해서 얻을 수 있는 결과
	- 데이터 자체가 내포하고 있는 **정보를 추출하는 방법**은 **아주 중요**
		- ex) 시험 점수로 석차를 구해 장학금을 지급할 경우
			- 정보 추출 방법을 **데이터 처리**(Data Processing) 또는 넓은 의미로 
			  **정보 처리**(information Processing) 라고 한다.
- **지식**
	- 기존 **정보를 활용**하여 **새로운 정보를 창출**할 수 있는 능력
	- **여러 분야의 지식이 많이 쌓이면** 의사결정을 잘 할 수 있는 **통찰력이 향상**됨
- **지혜**
	- 주어진 **상황을 빨리 인식**하고 **정확하게 정보**를 **처리**할 수 있는 능력
	- 지혜는 **지식을 바탕**으로 **전체 상황**을 **파악**하고 **신속하게 판단**할 수 있는 통찰력

#### \[ 정보 시스템 ]

- **조직체**에 **필요한 Data 를 수집 $\cdot$ 저장**해 두었다가 **필요 시 처리**해서 **의사결정**에 
  **유용한 정보를 생성**하고 **분배**하는 **수단**
- 사용하는 목적에 따라 **경영 정보 시스템**, **군사 정보 시스템**, **인사 행정 정보 시스템**, 
  **의사 결정 지원 시스템** 등이 있음

#### \[ 자료 처리 시스템 ]

- 정보 시스템이 사용할 자료를 처리하는 경영 시스템의 서브 시스템
- 처리 형태
	- ##### 일괄 처리 시스템
		- 시스템의 효율성을 최대한 높이기 위하여 **일정 시간 또는 일정량의 데이터**를 
		  **모아서 한 번에 처리**하는 시스템
		- 특징
			- **시스템 중심**의 자료 처리 방법
				- 시스템의 효율성과 자원의 활용 극대화를 우선시
			- **반환 시간**(Turn Around Time)이 **늦지만** 하나의 작업이 모든 **자원을 독점**하므로
			  **CPU 의 유휴 시간이 줄어듬**
			- 테이프와 같은 **순차 접근 방법**을 **사용하는 업무**에 **적합**
			- **단위 시간 당 처리**하는 **작업양이 많으므로** 시스템의 성능이 높음
				- 같은 시간동안 처리할 수 있는 작업(트랜잭션)의 수가 많다
			- **트랜잭션당 처리 비용이 적음**
				- 한 건의 트랜잭션을 처리하는 데 필요한 자원이 적다
		- 적용 파트
			- 급여계산
			- 회계 마감 업무
			- 세무 처리
			- 수도 $\cdot$ 전지요금 처리
			- 연말 결산 등
	- ##### 온라인 실시간 처리 시스템
		- **데이터 발생 시** 즉시, 또는 **데이터 처리 요구**가 있는 **즉시** 처리하여 
		  **결과를 산출**하는 방식
		- 온라인 처리 시스템이라고도 함
		- 특징
			- **사용자 중심**의 처리 방식
				- 사용자의 편의성과 응답 속도를 우선시
			- **처리 시간이 단축**되고, 처리 **비용이 절감**됨
		- 적용 파트
			- 우주선 운행이나 레이터 추척기
			- 핵물리학 실험 및 데이터 수집
			- 전화 교환장치의 
			- On-Line 업무 등 **시간에 제한을 두고 수행되어햐 하는 작업**에 사용
	- ##### 분산 처리 시스템
		- **지리적으로 분산**되어 있는 여러 대의 **컴퓨터**(프로세서)를 **통신 회선으로 연결**하여 **논리적을 하나의 시스템을 사용하는 것**처럼 운영하는 방식
		- 특징
			- 각 단말기나 컴퓨터 시스템은 고유의 OS 와 CPU, 메모리를 가짐
			- 시스템의 **구축이나, 운영이 복잡**한 반면 **신뢰성이 높고 확장이 용이**

#### \[ 데이터베이스의 정의 ]

- 데이터베이스는 **특정 조직의 업무를 수행**하는 데 **필요한 상호 관련된 데이터들**의 모임
- **같은 데이터**가 **다른 목적**을 가진 여러 응용에 **중복되어 사용**할 수 있다는 **공용의 개념**에 기초
	- 같은 데이터베이스에 다른 목적으로 사용을 할 수 있다
- ##### 통합데이터(Integrated Data)
	- 데이터베이스는 **같은 데이터**가 원칙적으로 **중복되어 있지 않음**
		- 통제된 중복
			- 효율성 증진의 위해서 데이터의 중복이 허용될 때도 있음
	- 데이터베이스는 **중복을 최소화**한 통합 데이터이다
- ##### 저장 데이터(Straged Data)
	- 자기 디스크나 테이프와 같이 **컴퓨터가 접근할 수 있는 저장 매체**에 **저장된 데이터**
	- 컴퓨터를 이용한 자료의 저장 관리가 당연시되고 있음
- ##### 운영 데이터(Operational Data)
	- 데이터베이스에 저장되어 있는 데이터는 운영 데이터이다.
	- 현재 시점에 필요(사용)한 데이터로, 데이터베이스는 매번 업그레이드 되어야 한다
	- **어떤 조직**이나 그 **고유의 기능**을 **가지고** 있고, 그 **기능을 수행**하기 위해 **반드시** 
	  **필요한 데이터**
	- 단순한 데이터의 집합이 아닌 그 **조직의 존재 목적**이나 **기능을 수행**하는 데 **없어서는 안 될 데이터**의 집합
	- 단순한 입 $\cdot$ 출력 데이터나 작업 처리 과정에서 **일시적으로 생성**된 
	  **임시 데이터**는 **운영 데이터로 인정되지 않음**
- ##### 공용 데이터(Shared Data)
	- 데이터 베이스는 공용 데이터이다
	- 어느 하나의 응용 프로그램이나 응용 시스템을 위한 데이터가 아닌 그 조직에 있는 여러 응용 시스템들이 **공동으로 소유**하고 **유지**하며, **이용**하는 공용 데이터
	- 여러 **사용자들이 서로 다른 목적**으로 **데이터베이스**를 **공동으로 이용**한다는 것을 뜻함
	- 데이터베이스는 일반적으로 그 **데이터 양의 규모가 대형화**되고 그 **구조가 복잡**하게 되는 것이 **보편적**이다

### < 데이터 베이스 시스템과 파일 시스템 >

#### \[ 파일 시스템 ]

- 영구적 데이터를 디스크에 저장하는 가장 보편적인 방법은 파일 시스템을 이용하는 것
- 1960년대부터 사용
- **파일**(데이터의 모임)을 저장 장치에 **저장하고 사용하기 위한 일종의 규칙**이나 **체계**
	- 파일의 이름을 붙이고 쉽게 파일에 접근할 수 있도록 배치를 신경 쓰는 등 파일과 관련된 기능을 수행하는 시스템
- OS 의 중요한 부분으로 테이터나 프로그램을 디스크에 쓰고 읽을 수 있도록 해주는 프로그램
- 파일 시스템에서는 정보를 OS 의 파일에 저장
- 각종 서류를 컴퓨터에 파일 단위로 저장하면, 회사의 데이터 관리 효율성과 활용성을 높일 수 있다
- 파일 시스템이서 파일은 **이용하는 프로그램 내에서 정의**하고 파일을 **접근하는 방식**이 
  응용 프로그램 내에 표현되므로 **응용 프로그램에 대한 의존도**가 **높다**
- ![[Pasted image 20250824145331.png]]
	
- 장점
	- **빠른 처리속도**
		- 운영체제가 제공하는 파일시스템을 사용하므로 처리 속도라 빠르다
	- **단순성**
		- 기능이 단순하기 때문에 단시간에 익힐 수 있으며 비용도 비교적 저렴
	- **저비용**
		- OS 에 내장되어 공급되기 때문에 별도의 구입, 유지보수, 교육, 훈련 등의 비용이 들지 않음
- 단점
	- **데이터 중복**
		- 데이터가 많은 파일에 중복되어 데이터 변경을 제어하는 것이 어렵기에 
		  **중복된 데이터 간에 불일치**가 발생
		- 중복된 데이터로 인한 **기억 공간 낭비**
	- **무결성**
		- 파일 내의 데이터가 만족시켜야 하는 무결성 제약 조건을을 명시하려면 
		  프로그래머가 직접 프로그래밍 언어를 사용하여 일일이 프로그램에 표현해야 
		  하므로 무경성 유지가 어려움
		- 새로운 제약 조건들을 추가하거나 기존의 제약 조건을 수정하는 것도 어려움
	- **보안**
		- 사용자의 **권한에 따른 세밀한 수준의 접근 제어를 시행**하기 **어려움**
	- **회복 기능**
		- 응용 프로그램에서 파일 내의 데이터를 수정하는 도중에 컴퓨터 시스템이 다운되어 재가동되었을 때 **데이터의 일관성을 복구하기 어려움**
	- **병행 제어**
		- 파일 시스템에서는 **병행 제어 기능이 없으므로** 동시에 **여러 사용자가 같은 자료**에 **접근**할 때 **갱신 유실 문제 등**의 오류가 발생하여 **일관성이 깨짐**
	- **유지보수 비용**
		- 파일의 구조가 응용 프로그램에 반영되어 있기 때문에 파일의 구조가 바뀌면 영향을 받는 모든 응용 프로그램들을 수정해야 한다
		- 응용 프로그램의 기능을 확장하려면 파일의 구조에 대한 요구사항이 바뀌므로 파일을 재조직 해야 한다
	- **데이터의 모델링 개념
		- 데이터의 의미와 데이터 간의 상호 관계를 나타내기 어렵다
	- 데이터 공유
		- 각 응용 프로그램마다 파일들을 갖고 있으며 데이터를 접근하는 응용 프로그램들이 여러 가지 프로그래밍 언어로 작성되어 데이터 공유가 제한됨
		- 

##### \[ DBMS(DataBase Management System) ]

- 데이터베이스 관리 시스템
- **파일시스템이 갖는 데이터의 종속성과 중복성의 문제를 해결**하기 위해 제안된 시스템
- 모든 응용 프로그램들이 데이터베이스를 **공용할 수 있도록 관리**
- 데이터베이스의 **구성, 접근방법, 유지관리**에 대한 **모든 책임**을 진다
- ![[Pasted image 20250824153958.png]]
	
- DBMS 필수 기능
	- **Define**(정의) 기능
		- 모든 응용 프로그램들이 요구하는 데이터 구조를 지원하기 위해 
		  DB에 저장될 데이터 형(Type)과 구조에 대한 정의, 이용방식, 제약조건 등을 
		  명시하는 기능
		- 데이터와 데이터의 관계를 명확하게 설명할 수 있어야 하며, 원하는 데이터 연산은 무엇이든 명세할 수 있어야 한다.
	- **Manipulation**(조작) 기능
		- 데이터 검색, 갱신, 삽입, 삭제 등을 체계적으로 처리하기 위해 사용자와 DB 사이의 인터페이스 수단을 제공하는 기능
	- **Control**(제어) 기능
		- DB 에 접근하는 갱신, 삽입, 삭제 작업이 정확하게 수행되어 데이터의 
		  무결성이 유지되도록 제어해야 한다
		- 정당한 사용자가 허가된 데이터에만 접근할 수 있도록 보안을 유지하고 권한을 검사할 수 있어야 한다
		- 여러 사용자가 DB 를 동시에 접근하여 데이터를 처리할 때 처리 결과가 항상
		  명확성을 유지하도록 병행 제어를 할 수 있어야 한다.
		
- 파일 시스템 방식과 DBMS 방식 비교
	- **파일 시스템 방식**
		- 데이터에 대한 물리적 접근만 조정
		- 동일한 파일에 두 개 이상의 프로그램이 동시 접근 불가능
		- 데이터가 비구조적
		- 중복성과 유지보수 비용이 높음
		- 어떤 프로그램이 기록한 데이터는 다름 프로그램에서 읽을 수 없는 경우가 많음
		- 데이터에 대한 접근은 미리 작성된 프로그램을 통해서만 가능
		- 각 응용 프로그램마다 파일이 따로 있으므로 데이터가 통합되어 있지 않음
	- **DBMS 방식**
		- 데이터에 대한 물리적 접근과 논리적 접근을 모두 조정
		- 동일한 데이터에 다수 이용자가 동시에 접근 가능
		- 데이터가 구조화되어 있음
		- 중복성과 유지보수 비용이 낮음
		- 접근 권한이 있는 모든 프로그래밍 데이터를 공유함
		- 질의어를 사용하여 데이터에 대한 융통서 있는 접근이 가능
		- 데이터가 중복을 배제하면서 통합되어 있음
#### \[ 데이터의 독립성 ]

- DBMS 를 사용하는 궁극적인 목적
	- **데이터의 논리적 구조나 물리적 구조가 변경**되더라도 **응용 프로그램의 영향을 받지 않는** 
	  데이터의 독립성(Data Independency) 을 확보하는것
- 데이터 독립성 구분
	- ##### 논리적(Logical) 데이터 독립성
		- 응용 프로그램과 DB를 독립시킴으로써, 데이터의 논리적 구조를 변경시키더라도 
		  응용 프로그램은 변경되지 않는다
	- ##### 물리적(Physical) 데이터 독립성
		- 응용 프로그램과 보조기억장치 같은 물리적 장치를 독립시킴으로써 
		  DB 시스템의 성능 향상을 위해 새로운 디스크를 도입하더라도 응용 프로그램에는 영향을 주지 않고 데이터의 물리적 구조만을 변경한다
- ##### 데이터의 종속성
	- 데이터의 구조가 변경되었을 때 응용 프로그램이 수정되어야 하는 현상

#### \[ 데이터의 중복성 ]

- 모든 응용 시스템들은 자기가 구현한 구조대로 데이터가 조직되어 있을 것을 요구
	- 그렇지 않으면 그 데이터를 처리할 수 없음
	- 파일 처리 시스템 또한 마찬가지
		- 그래서 **파일 시스템** 또한 **같은 정보**를 **여러 곳에 저장 가능**
- **데이터 중복성**
	- **한 시스템 내**에 **내용이 같은 데이터**가 **여러 군데 반복**되어 있는 것
- 데이터의 중복성의 문제
	- ##### 일관성(Consistency)
		- 중복된 데이터들 간의 내용이 전부 똑같지 않아 데이터의 일관성이 없어짐
		- DB는 모순성을 포함하게 되버림
	- ##### 보안성(Security)
		- 한 시스템 내에 존재하는 중복 데이터에 대해 같은 수준의 데이터 보안 적용이 안됨
			- 여러 곳에 존재 하는 중복 데이터에 동일한 수준의 보안 적용은 매우 어려움
	- ##### 경제성(Economics)
		- 동일한 데이터 중복 저장시 저장 수 만큼 저장 공간의 필요성이 증가
		- 중복된 데이터 하나의 수정으로 일관성을 위해 모든 데이터를 찾아서 수정해야하는 작업 비용이 증가
	- ##### 무결성
		- 데이터가 중복 저장되면 자연히 제어가 분산되어 데이터 무결성, 즉 데이터의 정확성을 유지하지 어려움
		- 데이터 수정시 동일 데이터를 모두 수정해야만 하는데, 하나라도 수정이 안될 경우 데이터의 무결성이 깨짐
		- 데이터 무결성은 데이터를 보호하고, 항상 정상인 데이터를 유지하는 것
		- 무결성은 정밀성, 정확성의 의미로 사용됨
			- DB 의 정확성을 보증한다는 말
		- 관계형 DB 에서 무결성
			- 데이터의 정확성과 일관성을 유지
			- 데이터에 결손과 부정합이 없음을 보증하는 것을 의미

##### \[ 무결성 ]

- **무결성 개념**
	- **DB에 저장된 데이터 값**과 그것이 표현하는 **현실 세계의 실제 값**이 **일치하는 정확성**을 의미
	- **무결성 제약조건**
		- **DB**에 저장된 데이터의 **정확성을 보장**하기 위해 **정확하지 않은 데이터가 DB 내**에 
		  **저장되는 것을 방지**하기 위한 제약조건
		- 제약조건에 위배되면 오류를 일으킴
- **무결성의 조건**
	- ##### NULL 무결성
		- 릴레이션의 특정 속성 값이 NULL이 될 수 없도록 하는 규정
		- **속성 무결성** 중의 하나
		- check, NOT NULL 등
	- ##### 고유(Unique) 무결성
		- 릴레이션의 특정 속성에 대한 각 튜플이 갖는 속성 값들이 서로 달라야 한다는 규정
			- = 하나의 도메인 내에선 중복된 값을 가질 수 없다
	- ##### 도메인(Domain) 무결성
		- 특정 속성의 값이 그 속성이 정의된 도메인에 속한 값이어야 한다는 규정
	- ##### 키(Key) 무결성
		- 하나의 릴레이션에는 적어도 하나의 키가 존재해야 한다는 규정(관계형 모델에서의 설명)
		- 하나의 릴레이션에는 같은 키 값을 갖는 튜플이 있어서는 안된다 **(교수님 정정)**
		- PK 와 UK 가 합친 형태로 Not Null 제약조건 적용 안됨
	- ##### 관계(Relationship) 무결성
		- 릴레이션에 어느 한 튜플의 삽입 가능 여부 또는 한 릴레이션과 다른 릴레이션이 
		  튜플들 사이의 관계에 대해 적절한지 여부를 지정한 규정
		- 외래키 무결성에 해당
	- 엔티티(Entity) 무결성
		- 고유 무결성 + 키 무결성
※ **릴레이션**
- 관계형 DB에서 데이터를 저장한 테이블
※ **NULL**
- DB 에서 아직 알려지지 않았거나 모르는 값으로 정보 부재를 나타내기 위해 사용
- 이론적으로 아무것도 없는 특수한 데이터
※ **도메인**
- 하나의 속성에서 취할 수 있는 더 이상 나누어 지지 않는 값들의 집합

#### \[ 동시 액세스 문제 ]

- 여러 자료를 여러 프로그램이 동시에 바꾸려고 할 때
	- 파일 처리 접근 방식
		- **동시 접근 제어 기능이 없어 자료의 값이 잘못될 수 있다**
	- DBMS
		- **동시 접근 제어 기능이 있어 동시에 자료에 접근할 경우 에러를 제어할 수 있음**

### < 데이터베이스의 특징 >
#### \[ 데이터베이스 시스템의 자기 기술성 ]

- DB 시스템의 주요 특징
	- DB 가 **DB 자체 뿐만 아니라 DB의 구조**와 **제약조건**에 대한 **완전한 정의까지 가짐**
- DB의 정의(**메타데이터**)
	- 메타 데이터는 DB에 속한 데이터 파일들에 대한 **논리적 구조**, 데이터 항목의 **타입**, **제약조건**, 데이터 파일의 **물리적 저장형식 및 접근 경로 등**을 **나타냄**
	- **시스템 카탈로그**(데이터 사전,DD: Data Dictionary)에 저장
- **자기 기술성**
	- 파일 시스템과 구별되는 특징
	- DBMS가 **데이터 삽입 및 삭제**를 데이터의 **구조적 종속없이 가능**하게 해줌
		- 파일 시스템에선 응용 프로그램이 파일 구조에 의존적이기 때문에 파일의 구조나 형식이 바뀌면 내부 코드도 수정해야 됨
		- DBMS 에선 시스템 카탈로그를 통해 특정 데이터만 읽어 오기 때문에, 카탈로그의 수정이 일어나도 응용 프로그램의 내부 코드를 수정할 필요가 없음

#### \[ 프로그램과 데이터의 격리 및 추상화 ]

- 프로그램과 데이터의 격리
	- 단일한 응용 프로그램 내에서 데이터를 개별적으로 관리하는 방식은 데이터 저장 구조 등이 변경되면 응용 프로그램도 수정되어야 한다
	- DB 는 '자기 기술성'을 가지므로 저장 구조 등을 수정하는 것이 응용 프로그램에 영향을 미치지 않는다
- **추상화**
	- 추상화란
		- **상세한 내용을 감추고 사용자가 이해하기 쉽게 단순화**하는 것
		- 크고 복잡한 문제를 단순화시켜 쉽게 해결하는 방법
	- **데이터 추상화**(Data Abstraction)
		- **프로그램 - 데이터 독립성**()
			- DBMS 에선 데이터 파일의 구조가 프로그램으로부터 분리됨
		- **프로그램 - 연산 독립성**
			- 데이터에 대한 연산을 DB 정의의 일부로 저장
	- **DMBS** 는 데이터가 어떻게 저장되고 연산들이 어떻게 구현되었는가에 관한 상세한 정보보단 **사용자에게 데이터에 관한 개념적인 표현을 제공**
	- **데이터 모델**
		- 사용자에게 데이터에 관한 개념적인 표현을 제공하는데 이용되는 데이터 추상화의 한 종류

#### \[ 다중 뷰 제공 ]

- ##### 뷰
	- **DB 테이블의 일부**이거나 **테이블로부터 유도되는 가상 데이터**
	- 뷰의 데이터는 **실제로 DB 에 저장되지 않음**
	- 사용자는 자신이 사용하는 데이터가 가상의 데이터인지 물리적으로 저장되어 있는 데이터인지는 알 필요 없음
- 다수의 관점으로 접근하는 사용자들에게 **저마다의 관점으로 데이터를 볼 수 있게 해주기 위해** DBMS 는 뷰를 통해 데이터를 제공한다

#### \[ 데이터 공유와 다수 사용자 트랜잭션 처리 ]

- DBMS 에서 **동시성 제어를 지원**하여 **다수의 사용자**가 **동일한 데이터를 동시에 변경**하는 경우에도 **데이터의 일관성을 보장**
- DBMS의 기본적인 기능은 **동시에 수행되는 트랜잭션들**이 **상호 방해를 받지 않고 정확**하고 
  **효율적으로 수행**되도록 **보장**

#### \[ 데이터베이스의 일반적인 특징 ]

- ##### 실시간 접근성(Real-Time Accessibility)
	- 수시적이고 비정형적인 질의(조회)에 대하여 **실시간 처리에 응답이 가능**해야 한다
	- 생성된 데이터를 즉시 컴퓨터에 보내어 그 처리 결과를 보고 다음 의사 결정에 바로 반영할 수 있게 하는 처리 방식
- ##### 계속적인 변화(Continuous Evolution)
	- DB의 상태는 동적
		- 새로운 데이터의 **삽입**(insert), **삭제**(delete), **갱신**(update)으로 **항상 최신의 데이터를 유지**해야 한다
- ##### 동시 공유(Concurrent Sharing)
	- DB 는 서로 다른 목적을 가진 여러 이용자들을 위한 것이므로 **다수의 사용자가 동시에** 
	  **같은 내용의 데이터를 이용할 수 있어야 한다**
- ##### 내용에 의한 참조(Content Reference)
	- DB에 있는 **데이터를 참조할 때** 데이터 레코드의 주소나 위치에 의해서가 아니라, 
	  **사용자가 요구하는 데이터 내용으로 데이터**를 찾음
